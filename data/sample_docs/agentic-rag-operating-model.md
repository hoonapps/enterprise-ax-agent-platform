# Agentic RAG 운영 모델

Agentic RAG는 질문 유형을 먼저 판단하고 검색 전략을 동적으로 선택한다.

질문 유형 예시는 다음과 같다.

- factual: 특정 정책이나 수치를 정확히 찾는 질문
- summary: 여러 문서를 요약하는 질문
- compare: 두 시스템이나 전략을 비교하는 질문
- action: 외부 시스템 실행이 필요한 질문
- risk: 보안, 장애, 규제, 운영 리스크를 검토하는 질문

검색 전략은 질문 유형에 따라 달라진다. 요약형 질문은 더 넓은 top-k 검색이 필요하고,
비교형 질문은 복수 출처의 균형이 중요하다. 실행형 질문은 검색 결과만으로 끝내지 않고
Tool 실행 가능 여부와 승인 필요 여부를 함께 판단해야 한다.

이 구조는 고객 맞춤형 AX 컨설팅, 사내 지식 검색, 업무 자동화 Agent에 모두 적용할 수 있다.
