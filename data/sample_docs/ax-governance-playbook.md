# AX 거버넌스 플레이북

기업용 LLM Agent는 사용자를 대신해 업무 시스템을 조회하거나 실행할 수 있으므로 권한과 감사 가능성이 중요하다.

운영 기준은 다음과 같다.

- 개인정보는 LLM context에 들어가기 전에 마스킹한다.
- 삭제, 결제, 송금, 인사 처리처럼 되돌리기 어려운 업무는 사람 승인 후 실행한다.
- 모든 Agent 실행은 요청자, 시나리오, 질문 유형, 사용 문서, 정책 판단, 결과 상태를 감사 이벤트로 남긴다.
- 근거 문서가 부족하면 답변을 확정하지 않고 담당자 확인 워크플로우로 넘긴다.
- Tool call은 read, write, approval 유형으로 구분하고, write 작업은 별도 정책을 적용한다.

이 원칙은 금융, 제조, SI, 유통 기업의 사내 생성형 AI 플랫폼에서 공통으로 필요하다.
