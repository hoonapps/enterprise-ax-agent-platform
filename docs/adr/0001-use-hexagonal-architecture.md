# ADR 0001: 헥사고날 아키텍처를 사용한다

## 상태

Accepted

## 배경

LLM 애플리케이션 예제는 HTTP 라우터, 프롬프트, SDK 호출, 벡터 검색, 정책 판단이 한 파일에 섞이는 경우가 많다.
이 방식은 데모는 빠르지만 기업용 Agent 백엔드로 설명하기 어렵다.

이 프로젝트는 한국 대기업 AX/AI Agent 직무 포트폴리오로 사용할 것이므로 다음을 보여줘야 한다.

- 업무 정책과 프레임워크의 분리
- LLM/Vector DB/MCP/n8n 교체 가능성
- API 없이도 테스트 가능한 유스케이스
- 감사로그와 권한 경계를 고려한 실행 구조

## 결정

실용적인 헥사고날 아키텍처를 사용한다.

- `domain`은 FastAPI, LangChain, Qdrant, OpenAI SDK를 import하지 않는다.
- `application`은 유스케이스와 orchestration을 담당한다.
- 외부 기술은 `adapter`에서 구현한다.
- 테스트는 라우터보다 유스케이스와 정책을 우선 검증한다.

## 결과

장점:

- 네트워크 없이 핵심 로직 테스트 가능
- LLM/Vector DB/Tool vendor 교체 가능
- 면접에서 백엔드 설계 의도를 설명하기 좋음
- 대기업 SI/ICT/금융/유통 AX 직무에 맞는 구조

트레이드오프:

- 단순 데모보다 파일 수가 많다.
- 라우터에 비즈니스 로직을 넣지 않도록 경계 관리가 필요하다.
